Universal Update Manager

Your friendly, set-it-and-forget-it tool for keeping your Linux system effortlessly up-to-date.

✨ Why You'll Love It

  • Effortless Updates Set a time and forget about it. Your system updates automatically every day!
  • Simple Graphical Menu No complex commands. A friendly menu guides you through every option.
  • Plays Well with Others Works on most major Linux distributions (Arch, Debian, Ubuntu, Fedora, etc.).
  • You're in Control Easily install, reconfigure the update time, or remove the service whenever you wish.
  • Peace of Mind Keep your system secure with the latest patches without lifting a finger.

🏁 How to Get Started

  1. Save the Script Save the main script code into a file named setup_autoupdate.sh.
  2. Make it Executable Open your terminal, navigate to where you saved the file, and run the following command. 
    chmod +x setup_autoupdate.sh
  3. Launch it! Now, simply run the script from your terminal to open the management menu. 
    ./setup_autoupdate.sh

📖 The Menu Options Explained

Install / Reconfigure Service: This is your starting point! Choose this to set up automatic updates for the first time or to change the daily update schedule later on. You'll be asked for your preferred time (e.g., 08:30 or 19:00).

Grant Sudo Permission: For the script to run updates without asking for your password every day, it needs permission. This option safely grants the ability to run *only* the package manager command without a password.

Remove Service: If you decide to go back to manual updates, this option cleanly removes all the script files, the schedule, and associated configurations.

Revoke Sudo Permission: This securely removes the password-free permissions you granted. It's the perfect way to tidy up if you're uninstalling the service.

🔧 For the Curious: How It Works

This tool is composed of a few simple, powerful components working together:

  • The Main Script (setup_autoupdate.sh): The friendly graphical menu you interact with, built using Zenity.
  • The Worker Script (AutoUpdatePackages.sh): This is the script that does the actual work of detecting your package manager and running the update commands. It's placed in ~/.local/share/autoupdate/.
  • The Scheduler (systemd Timers): Instead of a cron job, the script creates a modern systemd service and timer file in ~/.config/systemd/user/. This tells your system exactly when to run the updates each day.
  • The Log File (autoupdate.log): A simple log file is created in your home directory. You can check it anytime to see a history of when updates were run and what happened.
